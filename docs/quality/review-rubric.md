# Review Rubric

Last reviewed: 2026-05-03
Owner: Platform / Codex

## Priority Order

1. user-visible correctness
2. security and privacy
3. architecture boundaries
4. tests and verification
5. docs freshness
6. maintainability

## Required PR Notes

- what changed
- why it changed
- verification commands
- docs updated
- known follow-up debt

## Block Conditions

- broken `scripts/agent-task.sh verify`
- missing docs update for behavior/rule changes
- architecture scan failure
- secret leakage
- route or UI layer boundary violation
