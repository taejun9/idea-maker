# Design Docs Index

Last reviewed: 2026-05-03
Owner: Platform / Codex

## Purpose

This directory stores durable engineering and product design decisions. Use it for decisions that should survive beyond a single PR.

## Documents

- `core-beliefs.md`: product and engineering beliefs.

## Writing Rules

- Include `Last reviewed:` and `Owner:`.
- State context, decision, consequences, and enforcement.
- Link to execution plans when a decision came from implementation work.
- Keep decisions stable; use exec plans for temporary task state.

## Update When

- a core product assumption changes
- architecture direction changes
- a repeated review preference should become a rule
- a source of truth moves into or out of the repository
