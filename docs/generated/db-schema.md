## Status

PostgreSQL 18 is the selected database. Production migration files do not exist
yet; the API repositories currently create report history and source-index
tables idempotently when the Docker runtime starts or repository operations run.

## Tables

| Table | Purpose | Status |
| --- | --- | --- |
| `idea_reports` | generated report records and JSON report detail payloads | active |
| `source_observations` | normalized external source observations from approved live public source collectors | active |
| `report_sources` | link reports to source observations | planned |

## `idea_reports`

| Column | Type | Notes |
| --- | --- | --- |
| `id` | `uuid` | primary key, copied into the saved report payload |
| `idea` | `text` | normalized user idea for list queries |
| `locale` | `text` | request locale, default `ko-KR` |
| `research_requested` | `boolean` | true when the report used the search-and-organization path |
| `created_at` | `timestamptz` | report creation time; indexed descending for newest-first history |
| `report` | `jsonb` | full `IdeaReportResponse` payload used by the detail page, including generated Q1-Q4 intake answers and inferred or submitted Q5 answer |

Indexes:

- `idea_reports_pkey` on `id`
- `idea_reports_created_at_idx` on `created_at DESC`

Saved report deletion uses `DELETE FROM idea_reports WHERE id = $1`; there are no
dependent tables in the current MVP schema.

## `source_observations`

| Column | Type | Notes |
| --- | --- | --- |
| `id` | `bigserial` | primary key |
| `source_name` | `text` | normalized source or collector name |
| `source_url` | `text` | public source URL |
| `market` | `text` | `domestic_kr` or `overseas` |
| `category` | `text` | normalized source category |
| `title` | `text` | normalized source title |
| `summary` | `text` | normalized public source summary |
| `strengths` | `jsonb` | normalized strengths list |
| `weaknesses` | `jsonb` | normalized weaknesses list |
| `observed_date` | `date` | source observation date retained for freshness |
| `confidence` | `text` | `low`, `medium`, or `high` |
| `access_method` | `text` | original indexable access method, currently `live_http` |
| `embedding` | `jsonb` | local deterministic source vector derived from normalized public source fields |
| `indexed_at` | `timestamptz` | time the record was written to the source index |

Indexes:

- `source_observations_pkey` on `id`
- `source_observations_source_url_observed_idx` unique on `source_name`,
  `source_url`, `observed_date`, and `access_method`
- `source_observations_market_observed_idx` on `market, observed_date DESC`
- `source_observations_source_name_idx` on `source_name`

The source index stores normalized live public source records and local
deterministic source vectors only. It does not store raw user ideas, query
vectors, saved report payloads, fixture records, authenticated responses,
cookies, credentials, local files, provider embeddings, or vector ids.
