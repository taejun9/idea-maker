## Goal

Change the idea report flow so users do not manually enter Q1-Q4 intake answers. The system should generate those answers from the idea context, while images remain optional and are not required for report generation.

## Non-Goals

- Add a new external AI provider or research integration.
- Add image upload, image analysis, or image generation.
- Redesign unrelated report history or navigation screens.

## Assumptions

- "Q1~Q4를 입력받지말고 직접 생성" means the final report should still contain Q1-Q4 content, but it should be generated by the service instead of typed by the user.
- Images are optional and the flow must complete when no images are supplied.
- Q5 should be inferred from the entered idea by default, while staying editable in the UI.
- Deterministic fallback generation is acceptable because optional AI adapters must not be required for routine verification.

## Constraints

- Preserve frontend/backend layer boundaries.
- Backend routes stay thin; generation logic belongs in services.
- Frontend API calls stay in `apps/web/src/api/`.
- Validate report requests with Pydantic schemas.
- Run Docker-based checks if runtime behavior changes.

## Outcome

Changed the report creation flow so Q1-Q4 are generated by the backend from the idea and report organization context. The frontend no longer asks users to type Q1-Q4, infers Q5 from the entered idea, keeps Q5 editable, and submits only the selected Q5 override. The API also infers Q5 when clients omit it. Product, API contract, DB schema, security notes, and focused tests now describe the generated intake behavior.
