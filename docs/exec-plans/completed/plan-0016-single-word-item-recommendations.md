## Goal

When the user enters a single word in the idea input screen, recommend related concrete item ideas first, then generate the report for the selected recommended item.

## Non-Goals

- Do not add a new external recommendation provider.
- Do not add user accounts, saved recommendation history, or database persistence.
- Do not change the existing multi-word direct report flow unless needed to preserve compatibility.
- Do not weaken validation, lint, or test coverage.

## Assumptions

- A single word means the trimmed input contains exactly one whitespace-delimited token.
- Recommendations can be deterministic and generated by the backend service layer from the submitted keyword.
- The user should be able to choose a recommendation from the input screen and then continue into the existing report generation flow.
- Existing API routes should remain backward compatible for direct report generation.

## Constraints

- `apps/web` handles UI and client state only.
- `services/api` owns API schemas, orchestration, and domain service logic.
- Backend route handlers remain thin.
- Docker Compose is the source of truth for runtime tests.
- Keep new shared helpers local until reuse is proven.
